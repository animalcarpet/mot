Forms

See an example of DVSA form layout

Fieldsets and legends

  • a <fieldset> element must be used to group related form controls
  • the first element inside a fieldset must be a <legend> element which describes the group and acts as the section heading
  • <fieldset> elements can be nested. For example: dates, radio buttons and checkboxes
  • Do not use heading the elements <h1> - <h6> to create section headings within a form

Labels

  • a <label> element represents a question in the form
  • all form fields should be given labels
  • labels should be aligned above their fields
  • labels should be associated with form fields using the for attribute

Simple inputs

  • require a form-control class
  • to override the default (maximum) width add a size class (form-control-xxs - form-control-xl)
  • to force an input to always fill it's parent use the form-control-max class
  • use care when implementing <input type="number"/> as it exhibits inconsistent behaviour across browsers
  • if <input type="number"/> is used always apply min, max and step attributes
  • do not use the max-length attribute on form fields as this can lead to confusion. The maximum allowed length should be enforced through validation
  • for sensitive user data, for example bank details or username, add an autocomplete="off" attribute to ensure the browser doesn't remember the information

Drop-down lists

  • require a form-control class
  • to override the default (maximum) width add a size class (form-control-xxs - form-control-xl)
  • to force an input to always fill it's parent use the form-control-max class
  • where possible avoid using drop-down lists - use radio buttons or checkboxes instead
  • do not use a drop-down list to present the user with 2 options, instead use the Radio button pattern

Textareas

  • require a form-control class
  • to override the default (maximum) width add a size class (form-control-xxs - form-control-xl)
  • to force an input to always fill it's parent use the form-control-max class
  • to allow the user to resize the control vertically use the flexible-vert class
  • use the rows atribute to define height

Radio buttons

  • use radio buttons to let users choose a single option from a list
  • radio buttons must be contained within a <fieldset> element using a <legend class="form-label"> element as the associated "question"
  • to give the radio buttons their block form apply a class of block-label to the <label/>
  • radio buttons have been designed with large hit areas to be easier to select

Checkboxes

  • use checkboxes to let users select either on/off toggles or where multiple selections are possible
  • make it clear with words when users can select one or multiple options
  • checkboxes must be contained within a <fieldset> element using a <legend class="form-label"> element as the associated "question"
  • to give the checkboxes their block form apply a class of block-label to the <label/>
  • checkboxes have been designed with large hit areas to be easier to select

Flowing inputs

  • it may be necessary, for example in the case of address details, to make a number of inputs follow a single label
  • ensure all fields maintain their valid labels - by hiding the label with a class of visually-hidden
  • to reduce the spacing between elements apply a class of form-group-flowing to the <div class="form-group"> element

Dates

  • dates should be presented as 3 free format fields without using placeholder text
  • dates must be contained within a <fieldset> element using a <legend class="form-label"> element as the associated "question"
  • use <input type="text" pattern="[0-9]*"> as number exhibits inconsistent behaviour across browsers
  • ensure the form-group-day, form-group-month and form-group-year classes are correctly applied

Progressive engagement

  • where possible, if a question is dependant on a previous answer only reveal the related question once it becomes applicable
  • apply a data-target attribute to the input controlling the additional question with a value of the id to be revealed
  • if JavaScript is disabled the additional question will always show
